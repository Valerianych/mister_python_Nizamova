# Лабораторная работа 9

## Тема

`spec`, `spec_set` и `autospec`.

## Цель работы

Научиться создавать mock-объекты с ограничением интерфейса, чтобы тесты не проверяли несуществующие методы.

## Задание

1. Создать класс репозитория пользователей.
2. Создать сервис, который использует этот репозиторий.
3. В тестах использовать `create_autospec`.
4. Проверить вызовы методов и результат работы сервиса.

## Команда запуска

```bash
python -m unittest tests.test_lab_09_autospec -v
```

## Вывод

В работе показано, как mock с autospec помогает контролировать интерфейс зависимости.
