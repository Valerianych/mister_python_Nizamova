# Лабораторная работа 2

## Тема

Ветвления, граничные значения и неверные типы в unit-тестах.

## Цель работы

Написать функцию с понятным контрактом и покрыть её тестами так, чтобы были проверены нормальные значения, границы диапазона, пустые строки и неверные типы.

## Задание

1. Создать пакет `netutils` в папке `src`.
2. Добавить файл `src/netutils/ports.py`.
3. Реализовать функцию `parse_port(value)`.
4. Создать тестовый файл `tests/test_ports.py`.
5. Использовать `assertRaises` для проверки ошибок.
6. Использовать `subTest` для табличных проверок.

## Требования к `parse_port`

Функция должна принимать `int` или `str`.

Если передан `int`, значение должно быть в диапазоне от 1 до 65535 включительно.

Если передана строка, она сначала очищается через `strip()`, затем должна состоять только из цифр. После этого строка переводится в `int` и проверяется тот же диапазон.

Если тип другой, должна быть ошибка `TypeError`.

`bool` считается неправильным типом.

## Что проверяется в тестах

- границы `1` и `65535`;
- значения вне диапазона `0`, `-1`, `65536`;
- строки с пробелами;
- пустая строка и строка из пробелов;
- строки с буквами, плюсом и минусом;
- неправильные типы: `None`, список, словарь, дробное число, `True`, `False`.

## Команда запуска

```bash
python -m unittest tests.test_ports -v
```

## Вывод

В работе проверены разные ветви функции, граничные значения и ошибочные входные данные.
